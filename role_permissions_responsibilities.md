# 项目角色权限与责任说明书

## 1. 概述

本文档详细说明Oops-AI需求管理系统中各类角色的权限、责任和模块使用情况。系统支持四种核心角色：项目经理、产品经理、开发人员和测试人员，每种角色有明确的职责范围和系统权限边界。

## 2. 角色定义与层级关系

### 2.1 角色层级

| 角色 | 权限等级 | 描述 |
|------|----------|------|
| 项目经理 | 1 (最高) | 项目的最高负责人，拥有项目的全部权限 |
| 产品经理 | 2 | 负责需求管理和产品规划，拥有需求相关的全部权限 |
| 开发人员 | 3 | 负责开发实现，拥有开发相关的权限 |
| 测试人员 | 3 | 负责测试验证，拥有测试相关的权限 |

### 2.2 角色继承关系

- 项目经理拥有所有其他角色的权限
- 产品经理拥有部分项目经理的权限，以及开发和测试人员的部分权限
- 开发和测试人员拥有基础的系统访问权限

## 3. 角色权限详细说明

### 3.1 项目经理 (Project Manager)

#### 3.1.1 核心权限

| 模块 | 权限 | 说明 |
|------|------|------|
| 项目管理 | 全部 | 创建、编辑、删除项目；管理项目设置 |
| 成员管理 | 全部 | 添加/移除成员；设置成员角色；管理成员权限 |
| 需求管理 | 全部 | 创建、编辑、删除、分配需求；管理需求状态 |
| 工作流管理 | 全部 | 自定义需求状态流转规则；配置工作流 |
| 版本管理 | 全部 | 创建、编辑、删除版本；管理里程碑 |
| 团队协作 | 全部 | 评论、@提及、通知管理 |
| 数据分析 | 全部 | 查看项目仪表板；生成质量分析报告；导出数据 |

#### 3.1.2 具体操作权限

- **项目创建与管理**：创建新项目、编辑项目信息、删除项目、查看项目详情
- **成员管理**：邀请用户加入项目、移除项目成员、修改成员角色、设置成员权限
- **需求管理**：创建需求、编辑需求、删除需求、分配需求负责人、更新需求状态、设置需求优先级
- **工作流配置**：自定义需求状态、配置状态流转规则、设置角色权限
- **版本管理**：创建项目版本、设置版本里程碑、管理版本发布
- **团队协作**：添加评论、@提及团队成员、管理通知设置
- **数据分析**：查看项目指标、生成质量报告、导出项目数据

### 3.2 产品经理 (Product Manager)

#### 3.2.1 核心权限

| 模块 | 权限 | 说明 |
|------|------|------|
| 项目管理 | 只读 | 查看项目信息；查看项目成员 |
| 需求管理 | 全部 | 创建、编辑、删除、分配需求；管理需求状态 |
| 需求分析 | 全部 | 使用AI工具分析需求；生成用户故事；生成验收条件 |
| 质量分析 | 全部 | 查看需求质量评分；获取改进建议 |
| 团队协作 | 全部 | 评论、@提及、通知管理 |
| 数据分析 | 全部 | 查看项目仪表板；生成质量分析报告；导出数据 |

#### 3.2.2 具体操作权限

- **项目访问**：查看项目详情、查看项目成员列表
- **需求管理**：创建需求、编辑需求、删除需求、分配需求负责人、更新需求状态、设置需求优先级
- **需求分析**：使用模糊词识别、智能追问、用户故事生成器、验收条件生成器
- **质量分析**：查看需求质量评分、获取改进建议、生成质量报告
- **团队协作**：添加评论、@提及团队成员、管理通知设置
- **数据分析**：查看项目指标、生成质量报告、导出需求数据

### 3.3 开发人员 (Developer)

#### 3.3.1 核心权限

| 模块 | 权限 | 说明 |
|------|------|------|
| 项目管理 | 只读 | 查看项目信息；查看项目成员 |
| 需求管理 | 部分 | 查看需求；更新需求状态（开发相关）；添加评论 |
| 需求分析 | 只读 | 查看需求分析结果；查看用户故事；查看验收条件 |
| 团队协作 | 部分 | 评论、@提及 |

#### 3.3.2 具体操作权限

- **项目访问**：查看项目详情、查看项目成员列表
- **需求管理**：查看需求列表、查看需求详情、更新需求状态（从"approved"到"developing"，从"developing"到"testing"）、查看需求分析结果
- **团队协作**：添加评论、@提及团队成员、接收通知

### 3.4 测试人员 (Tester)

#### 3.4.1 核心权限

| 模块 | 权限 | 说明 |
|------|------|------|
| 项目管理 | 只读 | 查看项目信息；查看项目成员 |
| 需求管理 | 部分 | 查看需求；更新需求状态（测试相关）；添加评论 |
| 验收条件 | 全部 | 查看验收条件；生成测试用例；记录测试结果 |
| 团队协作 | 部分 | 评论、@提及 |

#### 3.4.2 具体操作权限

- **项目访问**：查看项目详情、查看项目成员列表
- **需求管理**：查看需求列表、查看需求详情、更新需求状态（从"testing"到"done"）、查看验收条件
- **测试管理**：查看验收条件、生成测试用例、记录测试结果、添加测试评论
- **团队协作**：添加评论、@提及团队成员、接收通知

## 4. 角色责任说明

### 4.1 项目经理责任

1. **项目规划与管理**
   - 制定项目计划和里程碑
   - 管理项目预算和资源
   - 确保项目按时交付

2. **团队管理**
   - 招募和管理项目团队
   - 分配任务和设置角色
   - 解决团队冲突

3. **沟通协调**
   - 与利益相关者沟通项目进展
   - 协调跨团队合作
   - 管理项目风险

4. **质量保证**
   - 确保项目质量符合标准
   - 管理项目变更
   - 批准需求和变更请求

### 4.2 产品经理责任

1. **需求管理**
   - 收集和分析用户需求
   - 编写和维护需求文档
   - 确保需求的完整性和清晰度

2. **产品规划**
   - 制定产品路线图
   - 确定需求优先级
   - 规划产品功能

3. **团队协作**
   - 与开发团队沟通需求
   - 与测试团队协作制定测试策略
   - 参与需求评审

4. **用户体验**
   - 确保产品满足用户需求
   - 收集用户反馈
   - 优化产品功能

### 4.3 开发人员责任

1. **需求实现**
   - 理解和分析需求
   - 设计和实现功能
   - 编写代码和单元测试

2. **代码质量**
   - 遵循编码规范
   - 进行代码审查
   - 修复bug

3. **项目协作**
   - 参与需求评审
   - 更新需求状态
   - 与团队成员沟通进展

4. **技术文档**
   - 编写技术文档
   - 维护代码注释
   - 记录技术决策

### 4.4 测试人员责任

1. **测试规划**
   - 制定测试计划
   - 设计测试用例
   - 准备测试环境

2. **测试执行**
   - 执行功能测试
   - 执行回归测试
   - 记录测试结果

3. **质量保证**
   - 发现和报告bug
   - 验证bug修复
   - 确保产品质量

4. **团队协作**
   - 与产品经理沟通测试需求
   - 与开发团队协作修复bug
   - 参与需求评审

- 配置项目工作流和通知设置

- 查看项目版本和里程碑

- 生成用户故事和验收条件
- 分配需求负责人和设置优先级
- 更新需求状态

- 更新需求状态（开发阶段）

- 添加评论和反馈

#### 5.2.3 测试人员使用
- 查看分配给自己的测试任务
- 更新需求状态（测试阶段）
- 查看验收条件和测试用例

- 参与需求讨论

### 5.5 数据分析模块

#### 5.5.1 项目经理和产品经理使用

- 导出项目数据和需求列表
- 分析项目进展和质量趋势

## 6. 权限管理与控制

### 6.1 权限检查机制

系统采用基于角色的访问控制（RBAC）机制，在以下层面进行权限检查：

1. **API层面**：所有API请求都经过权限验证中间件
2. **服务层面**：业务逻辑中包含权限检查
3. **数据库层面**：使用行级安全策略限制数据访问

### 6.2 权限提升机制

在特殊情况下，项目经理可以临时提升其他角色的权限：

- 为开发人员赋予临时的需求编辑权限
- 为测试人员赋予临时的需求查看权限
- 权限提升必须有明确的时间限制和原因记录

### 6.3 权限审计

系统记录所有权限相关的操作：

- 角色变更记录
- 权限提升记录
- 敏感操作日志

## 7. 角色协作流程

### 7.1 需求管理流程

1. **产品经理**：创建需求 → 分析需求 → 生成用户故事和验收条件 → 提交审核
2. **项目经理**：审核需求 → 批准需求 → 设置优先级
3. **开发人员**：认领需求 → 开发实现 → 更新状态为"开发中" → 完成后更新为"测试中"
4. **测试人员**：测试需求 → 记录测试结果 → 通过后更新为"已完成"/不通过则返回给开发

### 7.2 版本发布流程

1. **产品经理**：收集待发布需求 → 制定发布计划
2. **项目经理**：审核发布计划 → 协调资源
3. **开发人员**：完成版本开发 → 进行集成测试
4. **测试人员**：进行系统测试 → 出具测试报告
5. **项目经理**：批准发布 → 执行发布操作

- 定期培训和知识库更新

本文档定义了Oops-AI需求管理系统中各类角色的权限、责任和模块使用方式。通过明确的角色分工和权限控制，确保系统的安全使用和高效协作。系统支持灵活的角色配置，可以根据项目需要进行调整和扩展。
