# OOPS-AI 设计系统规范 V2.0

## 一、设计哲学

### 1.1 核心原则

| 原则 | 定义 | 实践方式 |
|------|------|----------|
| **AI优先，以人为本** | AI透明展示意图，用户掌握决策权 | AI建议以"轻量提示"呈现，用户可一键采纳或忽略 |
| **不离上下文** | 所有交互在当前视图完成 | 禁用模态弹窗，采用内联编辑、侧边抽屉、气泡卡片 |
| **极简与清晰** | 去除冗余，大量留白 | 单屏信息密度≤60%，核心任务≤3步完成 |
| **智能预见性** | AI主动预测下一步 | 输入时实时建议，操作后自动推荐后续动作 |

### 1.2 设计目标
```
✓ 沉浸式无中断工作环境
✓ AI辅助自然融入操作流
✓ 界面呼吸感，信息层级清晰
✓ 用户始终感觉"掌控全局"
```

## 二、交互模式革新

### 2.1 禁止使用的模式
```
❌ 模态弹窗
❌ 全屏遮罩层
❌ 强制确认对话框
❌ 跳转新页面完成子任务
❌ 打断式警告提示
```

### 2.2 推荐替代方案

| 传统模式 | 替代方案 | 实现方式 |
|----------|----------|----------|
| 模态弹窗 | 内联编辑 | 点击内容直接编辑，失焦自动保存 |
| 确认对话框 | 撤销机制 | 操作后显示Toast + 撤销按钮(5s内有效) |
| 全屏表单 | 渐进式表单 | 分步内联展示，当前步骤高亮 |
| 警告弹窗 | 状态栏提示 | 页面底部固定提示条，可展开详情 |
| 新建页面 | 侧边抽屉 | 右侧滑出面板，保持上下文可见 |

### 2.3 上下文内交互规范

#### 内联编辑
```
触发: 点击内容区域
状态: 内容转为输入框，背景微变
保存: 失焦自动保存 / Enter确认 / Esc取消
反馈: 成功时绿色边框闪烁(300ms)
```

#### 气泡卡片
```
触发: 点击图标/按钮
位置: 紧贴触发元素，不遮挡核心内容
尺寸: 最大宽度320px
关闭: 点击外部区域 / 再次点击触发器
```

#### 侧边抽屉
```
触发: 点击操作按钮
位置: 从右侧滑入
宽度: 400px / 480px / 600px (根据内容)
遮罩: 无遮罩或半透明(rgba(0,0,0,0.1))
关闭: 点击关闭按钮 / Esc键
```

## 三、AI 辅助交互规范

### 3.1 AI 建议呈现方式

#### 轻量提示
```
位置: 输入框下方 / 内容右侧
样式: bg-primary-50 border-l-2 border-primary-400
动画: 从上到下滑入(200ms)
操作: [采纳] [忽略] [查看详情]
```

#### 智能补全
```
触发: 用户输入时
样式: 灰色文字预览，Tab键采纳
位置: 紧跟光标位置
```

#### 操作推荐
```
位置: 操作完成后，底部出现
样式: 浮动卡片，半透明背景
内容: "您可能还需要..." + 3个推荐操作
自动消失: 8秒后淡出
```

### 3.2 AI 思考过程可视化

```
状态指示:
- 思考中: 脉动动画 + "AI正在分析..."
- 完成: 绿色勾号 + 结果摘要
- 可展开: 点击查看详细推理过程

展示位置:
- 右侧固定面板
- 不遮挡主内容区
- 可折叠收起
```

### 3.3 用户决策权保障

```
AI建议必须包含:
✓ 建议内容
✓ 建议理由(可折叠)
✓ [采纳] [修改后采纳] [忽略] 三个选项
✓ 忽略后记录偏好，优化后续建议
```

## 四、视觉设计规范

### 4.1 留白系统

```
页面留白率: ≥40%
卡片内部留白: ≥30%
元素间距倍数: 1.5x (基于4px单位)

具体数值:
- 页面边距: 24px (移动端) / 32px (桌面端)
- 区块间距: 24px
- 卡片内边距: 20px
- 元素间距: 12px
- 文字行间距: 1.6
```

### 4.2 信息密度控制

```
单屏原则:
- 核心操作 ≤ 3个
- 可视信息块 ≤ 5个
- 文字行数 ≤ 15行(大段文字需折叠)

表格原则:
- 可视列数 ≤ 6列
- 超出列隐藏在"更多"中
- 行高: 48px-56px
```

### 4.3 色彩系统(简化版)

```
主色系:
- Primary: #3b82f6 (AI智能、主要操作)
- Secondary: #6366f1 (辅助功能)

语义色:
- Success: #10b981 (完成、正确)
- Warning: #f59e0b (提醒、注意)
- Error: #ef4444 (错误、危险)

中性色:
- 背景: #fafafa (页面) / #ffffff (卡片)
- 边框: #e5e7eb
- 文字: #111827 (主) / #6b7280 (次) / #9ca3af (辅)

AI专属色:
- AI思考: #8b5cf6 (紫色系)
- AI建议背景: #f5f3ff
```

### 4.4 字体规范

```
字体族: 'Inter', 'PingFang SC', sans-serif

字号:
- H1: 24px (页面标题)
- H2: 18px (区块标题)
- H3: 16px (卡片标题)
- Body: 14px (正文)
- Caption: 12px (辅助)

字重:
- 标题: 600
- 正文: 400
- 强调: 500
```

## 五、组件设计规范

### 5.1 按钮

```
主要按钮:
- 样式: bg-primary-500 text-white rounded-lg
- 尺寸: h-10 px-5 text-sm
- 状态: hover:bg-primary-600 active:bg-primary-700

次要按钮:
- 样式: bg-white border border-gray-200 text-gray-700 rounded-lg
- 悬停: hover:bg-gray-50 hover:border-gray-300

文字按钮:
- 样式: text-primary-600 hover:text-primary-700
- 无边框无背景

按钮组:
- 间距: space-x-2
- 主操作在右侧
```

### 5.2 输入框

```
默认:
- 样式: border border-gray-200 rounded-lg px-3 py-2
- 高度: h-10
- 背景: bg-white

聚焦:
- 边框: border-primary-500
- 阴影: ring-2 ring-primary-100

内联编辑:
- 无边框
- 悬停显示虚线边框
- 点击转为输入框
```

### 5.3 卡片

```
基础卡片:
- 背景: bg-white
- 圆角: rounded-xl (12px)
- 阴影: shadow-sm
- 边框: border border-gray-100
- 内边距: p-5

悬停效果:
- 阴影: hover:shadow-md
- 边框: hover:border-gray-200
- 过渡: transition-all duration-200
```

### 5.4 表格(革新版)

```
替代传统表格的设计:
- 采用卡片列表形式
- 每行一个卡片，信息分层展示
- 操作按钮内联显示

如必须使用表格:
- 去除纵向边框线
- 仅保留行分隔线
- 表头背景透明
- 行高: 56px
- 悬停: hover:bg-gray-50
```

### 5.5 状态提示条

```
位置: 页面底部固定
样式: bg-white border-t border-gray-200 shadow-lg
高度: 48px (可展开至200px)
内容: 图标 + 消息 + 操作按钮
动画: 从底部滑入(300ms)
```

## 六、动效规范

### 6.1 核心动效

```
内联编辑:
- 进入: border-color变化(150ms)
- 退出: 淡出 + 缩小(200ms)

侧边抽屉:
- 进入: translateX(100% → 0) + fadeIn(300ms)
- 退出: translateX(0 → 100%)(200ms)

气泡卡片:
- 进入: scale(0.95 → 1) + fadeIn(150ms)
- 退出: fadeOut(100ms)

AI建议:
- 进入: translateY(-10px → 0) + fadeIn(200ms)
- 脉动: border-color闪烁(1s循环)
```

### 6.2 微交互

```
按钮点击: scale(0.98) → scale(1)(100ms)
输入聚焦: border-width 1px → 2px(150ms)
卡片悬停: translateY(-2px)(200ms)
成功反馈: 绿色边框闪烁(300ms)
```

## 七、响应式设计

### 7.1 断点

```
移动端: < 768px
平板: 768px - 1024px
桌面: > 1024px
```

### 7.2 布局适配

```
移动端:
- 单列布局
- 侧边抽屉全屏
- 底部固定操作栏
- 卡片堆叠

平板:
- 双列布局
- 侧边抽屉50%宽度
- 浮动操作按钮

桌面:
- 多列布局
- 侧边抽屉固定宽度
- 悬停显示详情
```

## 九、AI 编辑工具专用指令

### 9.1 快速生成指令

```
[创建内联编辑组件]
创建一个支持内联编辑的文本组件：
- 点击进入编辑模式
- 失焦自动保存
- 显示保存状态反馈
- 无模态弹窗

[创建AI建议组件]
创建AI建议气泡组件：
- 输入时实时显示
- 包含采纳/忽略/修改选项
- 可展开查看理由
- 动画流畅

[创建侧边抽屉]
创建右侧滑出抽屉：
- 从右侧滑入
- 无全屏遮罩
- 保持主内容可见
- 支持Esc关闭
```

## 十、设计检查清单

### 10.1 每次设计前检查

```
□ 是否使用了模态弹窗？→ 改用内联/抽屉
□ 是否打断用户工作流？→ 重新设计交互
□ 信息密度是否过高？→ 删减或折叠
□ AI建议是否可忽略？→ 添加忽略选项
□ 操作是否可撤销？→ 添加撤销机制
```

### 10.2 每次设计后检查

```
□ 核心任务是否≤3步完成？
□ 单屏信息块是否≤5个？
□ 是否有足够的留白？
□ 动画是否流畅不突兀？
□ 暗色模式是否正常？
```
